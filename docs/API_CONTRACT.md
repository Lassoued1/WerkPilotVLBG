# API Contract

## Global conventions

- Base path: `/api/v1`.
- JSON payloads except authorized file downloads.
- Authentication is required except login, refresh, and the two public
  password-reset endpoints.
- Pagination request parameters are `page`, `size`, and `sort`.
- Request timestamps are ISO-8601 with an offset; storage and canonical
  timestamp responses use UTC.
- IDs are UUID strings. Validation and authorization are backend-authoritative.
- OpenAPI 3.1 is generated by the backend and types the frontend client.

## Message and error contract

```json
{
  "timestamp": "2026-07-03T08:15:30Z",
  "status": 400,
  "errorCode": "CSV_VALIDATION_FAILED",
  "message": "The uploaded file contains invalid rows.",
  "path": "/api/v1/import-jobs/production-records",
  "details": [
    {
      "row": 14,
      "column": "units_produced",
      "value": "-5",
      "message": "Der Wert muss größer oder gleich null sein."
    }
  ]
}
```

Stable `errorCode` values and top-level diagnostics are English. The frontend
never displays the top-level `message`; it selects German catalog text using
`errorCode`. Dynamic CSV `details[].message` values are generated in German by
the backend. Tests assert codes rather than diagnostic wording, except for the
contractual German CSV details. This implements Sections 16.1 and 17.4; VAL-04
governs comment length/plain text, not language.

Supported codes include `AUTH_INVALID_CREDENTIALS`, `AUTH_TOKEN_EXPIRED`,
`ACCESS_DENIED`, `RESOURCE_NOT_FOUND`, `VALIDATION_FAILED`,
`CSV_MISSING_COLUMN`, `CSV_UNKNOWN_COLUMN`, `CSV_VALIDATION_FAILED`,
`IMPORT_DUPLICATE_FILE`, `BUSINESS_RULE_VIOLATION`, and `INTERNAL_ERROR`.
Never expose stack traces or internal class names.

## Shared response shapes

All modules reuse the Section 24.6 records.

Pagination:

```json
{
  "items": [],
  "page": 0,
  "size": 20,
  "totalElements": 0,
  "totalPages": 0
}
```

Nullable aggregate:

```json
{ "value": 4.82, "unit": "kWh_per_unit", "available": true }
```

```json
{
  "value": null,
  "unit": "kWh_per_unit",
  "available": false,
  "reason": "NO_UNITS_PRODUCED"
}
```

Aggregate/dashboard responses echo `appliedFilters`. `from` is inclusive and
`to` exclusive, matching `[period_start, period_end)`.

Asynchronous import status:

```json
{
  "jobId": "uuid",
  "status": "PROCESSING",
  "createdAt": "2026-07-05T12:00:00Z",
  "completedAt": null
}
```

`PROCESSING`, `COMMITTED`, `FAILED`, and `SUPERSEDED` are the complete job
status vocabulary. Upload polling ends at `COMMITTED` or `FAILED`;
`SUPERSEDED` is assigned later by correction or rollback.

## Explicit endpoint inventory

### Authentication

| Method | Path | Access | Contract |
| --- | --- | --- | --- |
| POST | `/auth/login` | Public | Return access token/profile/roles in JSON; set refresh cookie. |
| POST | `/auth/refresh` | Valid refresh cookie + CSRF header | Return a new access token and rotate refresh cookie. |
| POST | `/auth/logout` | Authenticated + CSRF header | Revoke and clear refresh cookie. |
| GET | `/auth/me` | Authenticated | Current profile and permissions. |
| POST | `/auth/password-reset-request` | Public, rate-limited | Enumeration-safe `202`; send mail when active account exists. |
| POST | `/auth/password-reset-confirm` | Public with valid reset token | Consume token, change password, revoke all refresh tokens. |
| POST | `/users/{id}/password-reset` | ADMIN | Trigger the same emailed reset flow; reveal no token/password. |

### Master data

| Method | Path | Access | Contract |
| --- | --- | --- | --- |
| GET, POST | `/factories` | Read authenticated; create ADMIN | Code/name master data; list excludes inactive by default. |
| GET, PUT, DELETE | `/factories/{id}` | Read authenticated; write ADMIN | Read/update/soft-delete. |
| GET, POST | `/production-lines` | Read authenticated; create ADMIN | Factory-linked line; code unique within factory. |
| GET, PUT, DELETE | `/production-lines/{id}` | Read authenticated; write ADMIN | Read/update/soft-delete. |
| GET, POST | `/machines` | Read authenticated; create ADMIN | Line-linked machine; code unique within its factory. |
| GET, PUT, DELETE | `/machines/{id}` | Read authenticated; write ADMIN | Read/update/soft-delete. |
| GET, POST | `/products` | Read authenticated; create ADMIN | Global product code/name and optional family. |
| GET, PUT, DELETE | `/products/{id}` | Read authenticated; write ADMIN | Read/update/soft-delete. |
| GET, POST | `/shifts` | Read authenticated; create ADMIN | Global shift code/name, times, and planned minutes. |
| GET, PUT, DELETE | `/shifts/{id}` | Read authenticated; write ADMIN | Read/update/soft-delete. |
| GET, POST | `/downtime-reasons` | Read authenticated; create ADMIN | Global downtime reason code/name. |
| GET, PUT, DELETE | `/downtime-reasons/{id}` | Read authenticated; write ADMIN | Read/update/soft-delete. |
| GET, POST | `/scrap-categories` | Read authenticated; create ADMIN | Global scrap category code/name. |
| GET, PUT, DELETE | `/scrap-categories/{id}` | Read authenticated; write ADMIN | Read/update/soft-delete. |

Master-data list endpoints use `page`, `size`, and optional
`includeInactive=false`. `GET /{resource}/{id}` remains historically
resolvable after soft delete and returns `active=false`.

### Global settings

| Method | Path | Access | Contract |
| --- | --- | --- | --- |
| GET | `/settings/global` | Authenticated | Return global settings, including `energyThresholdDelegationEnabled`. |
| PUT | `/settings/global/energy-threshold-delegation` | ADMIN | Toggle the global delegation flag and create `THRESHOLD_DELEGATION_CHANGED` audit when changed. |

`energyThresholdDelegationEnabled` defaults to `false`. ADMIN always passes
energy-threshold write authorization. ENERGY_MANAGER passes only while this
flag is `true`; other roles are denied.

### Threshold administration

| Method | Path | Access | Contract |
| --- | --- | --- | --- |
| GET | `/thresholds` | Authenticated | Paginated list of active thresholds by default; optional `metricKey`, `scopeType`, `includeInactive`, `page`, and `size`. |
| GET | `/thresholds/{id}` | Authenticated | Return one threshold, including inactive records. |
| POST | `/thresholds` | ADMIN; ENERGY_MANAGER for energy metrics only when delegation is ON | Create threshold and append `THRESHOLD_CHANGED`. |
| PUT | `/thresholds/{id}` | ADMIN; ENERGY_MANAGER for energy metrics only when delegation is ON | Replace threshold definition and append `THRESHOLD_CHANGED`. |
| DELETE | `/thresholds/{id}` | ADMIN; ENERGY_MANAGER for energy metrics only when delegation is ON | Soft-disable threshold and append `THRESHOLD_CHANGED`. |

Supported `metricKey` values are `ENERGY_KWH`, `ENERGY_PER_UNIT`,
`OUTPUT_PER_HOUR`, `AVAILABILITY`, `DOWNTIME_MINUTES`, and `SCRAP_RATE`.
Supported `scopeType` values are `GLOBAL`, `FACTORY`, `LINE`, `MACHINE`,
`PRODUCT`, and `SHIFT`. `GLOBAL` requires `scopeId=null`; all other scopes
require an existing master-data UUID. At least one of `minValue` or `maxValue`
is required, and `minValue <= maxValue` when both are present. `severity` is
`WARNING` or `CRITICAL`.

### Audit

| Method | Path | Access | Contract |
| --- | --- | --- | --- |
| GET | `/audit-events` | ADMIN | Paginated newest-first audit query. |

Supported query parameters are `page`, `size`, `eventType`, `actorUserId`,
`targetUserId`, `from`, and `to`. `from` is inclusive and `to` is exclusive.
Responses include `id`, `eventType`, `actorUserId`, `targetUserId`,
`occurredAt`, `details`, and nullable `traceId`.

### Imports

| Method | Path | Access |
| --- | --- | --- |
| POST | `/import-jobs/production-records` | ADMIN, PRODUCTION_MANAGER |
| POST | `/import-jobs/energy-measurements` | ADMIN, ENERGY_MANAGER |
| POST | `/import-jobs/downtime-records` | ADMIN, PRODUCTION_MANAGER |
| POST | `/import-jobs/scrap-records` | ADMIN, PRODUCTION_MANAGER |
| GET | `/import-jobs` | ADMIN and authorized managers |
| GET | `/import-jobs/{id}/errors` | ADMIN and authorized managers |
| POST | `/import-jobs/{id}/correction` | ADMIN; multipart replacement for a COMMITTED job |
| POST | `/import-jobs/{id}/rollback` | ADMIN; mandatory reason; no replacement |

Uploads return a `PROCESSING` job immediately. A correction uses the original
job type/template and commits atomically. Every active-data query includes only
rows whose job is `COMMITTED`.

### Production, KPIs, and analytics

| Method | Path | Access |
| --- | --- | --- |
| GET | `/production/kpis` | Authenticated per product permissions |
| GET | `/production/records` | Authenticated per product permissions |
| GET | `/dashboard/summary` | All roles |
| GET | `/energy/kpis` | ADMIN, PRODUCTION_MANAGER, ENERGY_MANAGER, VIEWER |
| GET | `/downtime/pareto` | ADMIN, PRODUCTION_MANAGER, VIEWER |
| GET | `/quality/scrap-rate` | ADMIN, PRODUCTION_MANAGER, VIEWER |
| GET | `/anomalies` | All roles |
| GET | `/anomalies/{id}` | All roles |
| PATCH | `/anomalies/{id}/status` | Authorized managers/admin; technician limited |
| POST | `/anomalies/rerun` | ADMIN |

`POST /anomalies/rerun` accepts `from`, `to`, and optional `factoryId`,
`lineId`, `machineId`, `productId`, `shiftId`. It returns created,
superseded, unchanged, and detected counts. Reruns use fixed detection identity;
unchanged reruns are no-ops, changed results supersede and link a NEW successor,
and disappeared results supersede without successor.

Ticket detail/assignment, report history, CSV evidence, and audit views are
required by the normative screens/workflows. Freeze their exact DTOs and paths
in OpenAPI before implementation.

## Filter contract

Use shared typed filters where applicable: `from`, `to`, `factoryId`,
`lineId`, `machineId`, `productId`, `shiftId`, `anomalyType`, `severity`,
`anomalyStatus`, `ticketStatus`, `priority`, `assigneeId`, `reasonId`, and
`scrapCategoryId`. Normalize time zones and echo effective filters.

## Contract completion gate

Before implementing a public endpoint, define and test its request, response,
validation, authorization, pagination, errors, and source references; generate
OpenAPI/frontend types; then run:

```powershell
cd backend
.\mvnw.cmd clean verify
cd ..\frontend
npm ci
npm run lint
npm test -- --run
npm run build
```
